## 核心功能

### 1. 趨勢追蹤策略（Trend Following Strategy）

本系統的核心策略，結合多重技術指標與多時間框架確認，有效過濾假信號：

- **MACD 交叉信號**：捕捉趨勢啟動的關鍵時刻
- **RSI 過濾器**：避免在超買/超賣區域追單，降低逆勢交易風險
- **成交量確認**：確保進場時有足夠的市場動能支撐
- **多時間框架確認**：以 15 分鐘為主要交易時間框架，搭配 1 小時線確認大方向，避免逆勢操作
- **ATR 動態止損/止盈**：
  - 止損：1.8 倍 ATR（根據市場波動度自動調整）
  - 止盈：2.5 倍 ATR
  - 風險報酬比：1:1.39

### 2. 網格交易策略（Grid Trading Strategy）— Pro 方案

專為盤整行情設計，趨勢策略無用武之地時自動切換：

- **自動偵測支撐/壓力位**：系統自動識別價格區間
- **智慧網格佈局**：在區間內自動建立多層買賣掛單
- **ADX 自動策略切換**：根據 ADX 指標判斷市場狀態，自動在趨勢策略與網格策略之間切換
- **適應盤整市場**：在橫盤震盪中持續賺取價差收益

### 3. 風險管理系統

交易中最重要的不是賺多少，而是控制虧損。本系統內建多層風控機制：

| 風控機制 | 說明 |
|----------|------|
| **單筆風險上限** | 每筆交易最多承擔帳戶 2% 的風險 |
| **最大回撤保護** | 帳戶回撤達 12% 時自動停止交易 |
| **連續虧損偵測** | 偵測到連續虧損時自動縮減倉位大小 |
| **每日虧損上限** | 單日虧損達 5% 時暫停交易至隔日 |

### 4. 回測引擎

上線前先用歷史數據驗證策略表現：

- 支援 30 天以上的歷史數據回測
- 詳細績效報告：勝率、盈虧比、最大回撤、利潤因子等
- 逐筆交易紀錄，清楚了解每筆交易的進出場邏輯
- **先回測，再實盤** — 用數據說話，不靠感覺

### 5. Telegram 即時通知 — Pro 方案

交易訊號即時推送到手機：

- 開倉/平倉通知
- 止損/止盈觸發通知
- 每日績效摘要
- 異常狀況警示

### 6. 即時儀表板 — Pro 方案

精美的終端介面，一目了然掌握所有資訊：

- 帳戶餘額與損益狀態
- 當前持倉與未實現盈虧
- 市場分析指標即時更新
- 交易歷史紀錄

## 安裝步驟

### 步驟一：下載專案

購買後下載專案壓縮檔並解壓縮，或使用 Git：

```bash
git clone <你的專案網址>
cd crypto-trading-bot
```

### 步驟二：安裝相依套件

```bash
pip install -r requirements.txt
```

### 步驟三：設定環境變數

將範例檔案複製一份並填入你的 API 金鑰：

```bash
cp .env.example .env
```

編輯 `.env` 檔案，填入以下資訊：

```env
BINANCE_API_KEY=你的幣安API金鑰
BINANCE_API_SECRET=你的幣安API密鑰
TELEGRAM_BOT_TOKEN=你的Telegram Bot Token（Pro方案）
TELEGRAM_CHAT_ID=你的Telegram Chat ID（Pro方案）
```

### 步驟四：調整設定檔

編輯 `config.yaml`，根據你的需求調整參數（詳見下方設定說明）。

### 步驟五：執行回測

**強烈建議先執行回測確認策略表現，再進行實盤交易：**

```bash
python run_backtest.py
```

### 步驟六：啟動自動交易

確認回測結果滿意後，啟動實盤交易：

```bash
python main.py
```

## 設定說明

`config.yaml` 主要設定項目：

```yaml
# 交易對設定
symbols:
  - BTC/USDT
  - ETH/USDT

# 策略參數
strategy:
  timeframe: "15m"           # 主要時間框架
  confirmation_tf: "1h"      # 確認時間框架
  macd_fast: 12
  macd_slow: 26
  macd_signal: 9
  rsi_period: 14
  volume_threshold: 1.2      # 成交量門檻倍數

# 風險管理
risk:
  max_risk_per_trade: 0.02   # 單筆最大風險 (2%)
  max_drawdown: 0.12         # 最大回撤上限 (12%)
  daily_loss_limit: 0.05     # 每日虧損上限 (5%)
  atr_sl_multiplier: 1.8     # ATR 止損倍數
  atr_tp_multiplier: 2.5     # ATR 止盈倍數

# 網格交易（Pro 方案）
grid:
  enabled: true
  adx_threshold: 25          # ADX 低於此值切換為網格模式
  grid_levels: 5             # 網格層數
```

你可以根據自己的風險偏好與資金規模調整以上參數。

## 常見問題 FAQ

### Q：我是交易新手，適合使用嗎？

A：適合。本系統設計的初衷就是幫助交易者排除情緒干擾，用系統化的方式交易。安裝步驟簡單明瞭，設定檔也有詳細註解。建議先用回測功能熟悉系統，再以小額資金開始實盤。

### Q：最低需要多少資金？

A：技術上 $100 USDT 即可運行，但建議至少 $500 USDT。資金越充足，風控機制越能發揮作用，單筆虧損對帳戶的影響也越小。

### Q：支援哪些交易對？

A：支援幣安 USDT 永續合約上所有交易對。預設配置為 BTC/USDT 和 ETH/USDT，你可以在 `config.yaml` 中自行新增。

### Q：需要一直開著電腦嗎？

A：是的，程式需要持續運行才能監控市場和執行交易。建議使用雲端伺服器（如 AWS、GCP 或便宜的 VPS）來確保 24/7 不中斷。

### Q：API Key 安全嗎？

A：你的 API Key 儲存在本地的 `.env` 檔案中，不會上傳到任何地方。建議在幣安設定 API 時：
- 僅開啟「合約交易」權限
- 設定 IP 白名單
- 不要開啟「提幣」權限

### Q：回測結果跟實盤會一樣嗎？

A：回測結果僅供參考。實盤交易會受到滑點、延遲、流動性等因素影響，實際表現可能與回測有所差異。這也是為什麼我們建議先以小額資金測試。

### Q：Basic 和 Pro 方案的差別是什麼？

A：Basic 方案包含核心的趨勢追蹤策略、回測引擎和風險管理系統。Pro 方案額外提供網格交易策略、Telegram 即時通知、即時儀表板，以及優先技術支援。

### Q：可以同時交易多個交易對嗎？

A：可以。在 `config.yaml` 中加入多個交易對即可。系統會獨立追蹤每個交易對的信號與持倉。

### Q：如何取得幣安 API Key？

A：登入幣安 → API 管理 → 建立 API → 記錄 API Key 和 Secret Key → 開啟合約交易權限 → 建議設定 IP 白名單。
